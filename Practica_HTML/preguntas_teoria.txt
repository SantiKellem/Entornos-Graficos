Ejercitacion 1 

1)
Qué es HTML, cuando fue creado, cuáles fueron las distintas versiones y cuál es la última?

HTML Es un estándar a cargo del World Wide Web Consortium (W3C) o
Consorcio WWW (creado en 1990), organización dedicada a la estandarización de casi
todas las tecnologías ligadas a la web, sobre todo en lo referente a su
escritura e interpretación.
Se considera el lenguaje web más importante siendo su invención
crucial en la aparición, desarrollo y expansión de la World Wide Web (W
WW).
Las distitas versiones fueron: HTML 2.0, HTML 3.2, HTML 4.0, HTML 4.01, XHTML
 y la mas nueva HTML 5

2)
 ¿Cuáles son los principios básicos que el W3C recomienda seguir para la creación de documentos
con HTML?

Los principios basicos que se recomiendan por parte de W3C son:
Semantica: Tener una estructura clara y significativa en el documento
Presentación: Separar el contenido de HTML y CSS y definir el estilo y el diseño
Compatibilidad con Navegadores: El codigo debe ser compatible con una amplia variedad de navegadores web
Rendimiento: Optimizar el tamaño de los archivos asi reduce el tiempo de carga de la pagina
Internacionalización: Que el documento pueda ser entendido y utilizado por personas de diferentes culturas

3)
En las Especificaciones de HTML, ¿cuándo un elemento o atributo se considera desaprobado? ¿y
obsoleto?

Segun las especificaciones de HTML un elemento se considera desaprobado cuando modifica el estilo,
pues del estilo de la pagina se encarga CSS. Por otro lado, un elemento se considera obsoleto cuando
los navegadores web ya no lo interpretan.

4)
Qué es el DTD y cuáles son los posibles DTDs contemplados en la especificación de HTML 4.01?

DTD o Document Type Declaration es la declaracion del tipo de documento y los posibles DTDs contemplados en la especificacion de HTML 4.01 son: 
DTD HTML 4.01 Estricto (Strict DTD), DTD HTML 4.01 Transicional (Transitional DTD) y DTD HTML 4.01 para Documentos con Marcos (Frameset DTD)

5)

Qué son los metadatos y cómo se especifican en HTML?

Los metadatos son informacion de un documento agregada sobre el propio contenido del documento, el elemento meta puede usarse para identificar 
propiedad de un documento como: especificar el autor, fecha de caducidad y listas de palabras claves

c)
<img src="" alt="lugar imagen" id="im1" name="im1" width="32" height="32" longdesc="detalles.htm" />

img es una etiqueta propia de imagenes, src es un atributo para poner la direccion de esa imagen tanto local como externa, alt texto que se muestra en lugar de la imagen

d)
<meta name="keywords" lang="es" content="casa, compra, venta, alquiler" />
<meta http-equiv="expires" content="16-Sep-2019 7:49 PM" />

meta name definido previamente, su valor "keywords" sirve para establecer las palabras claves de la pagina (palabras que favorecen a nuestra pagina en el momento de la busqueda)
lang es un atributo que sirve para definir el idioma siendo "es" = español, content es un atributo que da el contenido de las palabras claves, http-equiv tiene control sobre los exploradores,
y son utilizadas para refinar la información y dar instrucciones al explorador que las esta leyendo. Y su valor "expires" indica al explorador cuando acaba la vida de una página, 
limitando el uso de una la misma a una fecha y una hora determinada. Es una Meta útil siempre que el contenido de la web sea perecedero. Content es la fecha de expiracion

e)
<a href="http://www.e-style.com.ar/resumen.html" type="text/html" hreflang="es" charset="utf-8"
rel="help">Resumen HTML </a>

a es una etiqueta que se utiliza para poner enlances en la página; href es un atributo obligatorio en el que se escribe la dirección web de la página a la que queremos redirigir,
http://www.e-style.com.ar/resumen.html en este caso; type sirve para indicar el tipo de archivo del link, que en el caso del ejemplo es un archivo html; hreflang nos permite especificar 
(solo de manera informativa, no funcional) el lenguaje de la página a la que estamos redirigiendo, español en este caso; charset se utiliza para especificar el tipo de codificación de la
página linkeada, siendo utf-8 el estándar en la actualidad; rel se usa para definir la relación entre la página linkeada y nuestra propia página y el atributo help indica que la página puede
tener contenido sensible para el usuario.

table se utiliza para indicar que el contenido de la etiqueta tendrá formato de tabla; width sirve para indicar el ancho de la tabla; summary representa un resumen de los contenidos de la tabla o
una explicación de como navegarla; caption se usa para escribir el título o descripción de la tabla, siendo align un atributo que permite ubicar ese texto en el lugar especificado
